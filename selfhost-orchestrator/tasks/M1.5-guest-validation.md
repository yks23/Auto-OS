## 你必须做的事

### 阶段 A：跑通 riscv64

1. 在你的 worktree 内：
   ```bash
   bash scripts/integration-build.sh ARCH=riscv64
   cd tests/selfhost && make ARCH=riscv64 && cd ../..
   sudo bash scripts/run-tests-in-guest.sh ARCH=riscv64 TIMEOUT=300
   ```
2. **如果 QEMU 启动失败、kernel panic、rootfs 加载失败、串口连不上**：
   - **先尝试自己修**：可能要调 `qemu-run-kernel.sh` 的参数（machine type、bios、bus 等），可以参考 `tgoskits/os/StarryOS/configs/qemu/qemu-riscv64.toml` 的配置和 `tgoskits/os/StarryOS/scripts/ci-test.py` 的做法
   - **不能修的**就停下来，把完整日志（`.guest-runs/riscv64/qemu.log` 与 `results.txt`）放进输出，让 Director 处理
3. 最少要让 **kernel boot 到 BusyBox shell prompt**（`starry:~#`），然后 `sh /opt/run-tests.sh` 跑出 `===SELFHOST-SUMMARY total=31 pass=N fail=M skip=K===` 一行。
