## 残留 bug 推测

bisect 简单形式 OK 但完整组合 hang 的差异：
- bisect 子进程是 immediate `_exit(42)` 或 `execve("/bin/echo")`（小、快、无 pipe）
- 测试用 `pipe + dup2 + execve + close pfd[1]` + parent `read(pfd[0])` 然后 `waitpid`

可能根因（按嫌疑）：
1. **read(pipe) 阻塞与 child execve+exit 之间有第二条 lost wakeup**：父进程 read pipe 等数据，child execve 把 stdout dup2 到 pfd[1]，写入后 exit，但 pipe writer 关闭事件没正确 wake 父 read
2. **dup2 + close(pfd[1]) 后 fd table refcount 错**：execve 时 close-on-exec 关闭了 dup2 后的 stdout，pipe 再没人写
3. **waitpid 的另一条等待路径**（不是 child_exit_event 而是别的）

需要更深 bisect。
